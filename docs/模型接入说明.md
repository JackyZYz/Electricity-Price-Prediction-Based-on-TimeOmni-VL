## 1. 理解当前抽象接口

所有 backbone 必须实现 `BackboneAdapter`：

```python
class BackboneAdapter(ABC):
    def load(self, model_path, device, **kwargs) -> None
    def understand(self, image, prompt, ...) -> str
    def generate(self, source_image, prompt, cot=None, ...) -> Image.Image
    def train_step(self, batch) -> Dict[str, float]
    def save_checkpoint(self, path) -> None
    def load_checkpoint(self, path) -> None
    def get_trainable_params(self) -> Dict[str, Any]
```

当前实现：
- `MockAdapter` — 占位，已跑通数据流
- `BagelAdapter` / `JanusAdapter` — 只有空壳，抛出 `NotImplementedError`

## 3. 推荐的最小接入路径

如果你想快速验证真实模型，建议按这个顺序：

### 阶段 A：仅做理解推理（最容易）

1. 下载 Janus-Pro-1B（显存需求低）
2. 实现 `JanusAdapter.load()` 和 `JanusAdapter.understand()`
3. 用 `python -m timeomni_vl.cli.infer --task understand` 测试
4. 确认模型能看懂 TS-image 并回答 QA

### 阶段 B：仅做生成推理

1. 实现 `JanusAdapter.generate()`
2. 用 `python -m timeomni-vl.cli.infer --task forecast` 测试
3. 确认能生成 TS-image，再经 I2TS 解码出价格曲线

### 阶段 C：真实训练

1. 在 Adapter 内实现：
   - 模型加载
   - LoRA 配置
   - optimizer + scheduler
   - `train_step` 中的前向/损失/反向
2. 调整 `configs/train.yaml`：
   - 减小 `batch_size`（Janus 7B 至少需 24GB+ 显存）
   - 启用 `lora_enabled: true`
   - 减少 `num_epochs`，先跑 1-2 个 epoch 验证
3. 运行 `python -m timeomni_vl.cli.train --config configs/train.yaml`
